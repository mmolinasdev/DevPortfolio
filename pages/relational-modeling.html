Diseño de Bases de Datos con el Modelo Relacional

Modelo Relacional Esquemas Claves Relaciones

Descripción del Proyecto

Diseño de esquemas de bases de datos relacionales, escogiendo 2 de 3 casos de estudio propuestos: una aerolínea, una biblioteca y un sistema de urgencias hospitalarias. Aplicando los conceptos del Modelo Relacional según Garcia-Molina, Ullman y Widom.

Ejercicios Desarrollados

Aerolínea 7 tablas, 8 FK Gestión de vuelos, pilotos y tripulación
Biblioteca 6 tablas, 5 FK Sistema de préstamos y autores

Conceptos Aplicados

Relaciones (Relations): Tablas del modelo relacional
Tuplas (Tuples): Filas de datos en las tablas
Atributos (Attributes): Columnas con tipos de datos
Claves Primarias: Identificadores únicos
Claves Foráneas: Referencias entre tablas
Relaciones 1:N: Uno a muchos
Relaciones N:M: Muchos a muchos
Integridad Referencial: Consistencia de datos

Decisiones de Diseño Clave

  • Normalización: Eliminación de redundancia mediante separación de entidades
  • Tablas Intermedias: Resolución de relaciones muchos-a-muchos
  • Claves Naturales: Uso de identificadores del dominio (ISBN, cédula, número de vuelo)
  • Tipos de Datos Apropiados: string, integer, date, time, boolean según el contexto
  • Nomenclatura en Inglés: Siguiendo estándares profesionales internacionales
  • Integridad Referencial: Foreign Keys aseguran consistencia entre tablas relacionadas

Tipos de Relaciones Implementadas

1:N (Uno a Muchos)

Una entidad se relaciona con múltiples instancias de otra entidad. Ejemplo: Una base tiene muchos aviones, una editorial tiene muchos libros.

Ejemplos en ejercicios: Bases → Aviones, Bases → Pilotos, Editorial → Libros
N:M (Muchos a Muchos)

Múltiples instancias de una entidad se relacionan con múltiples instancias de otra. Requiere tabla intermedia. Ejemplo: Pilotos ↔ Vuelos, Libros ↔ Autores.

Ejemplos en ejercicios: Pilotos ↔ Vuelos, Tripulación ↔ Vuelos, Libros ↔ Autores

Tipos de Datos Utilizados

string → Cadenas de texto (nombres, códigos, direcciones)
integer → Números enteros (años, horas, niveles)
date → Fechas (YYYY-MM-DD)
time → Horas (HH:MM:SS)
boolean → Verdadero/Falso (true/false)
