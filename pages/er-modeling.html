Modelo Entidad-Relación

Diseño Conceptual de Bases de Datos

Modelo ER Entidades Atributos Relaciones

Descripción del Proyecto

Diseño conceptual de bases de datos utilizando el Modelo Entidad-Relación (ER), escogiendo 2 de 3 casos de estudio propuestos: una aerolínea, una biblioteca y un sistema de urgencias hospitalarias. El trabajo aplica los conceptos fundamentales del modelado conceptual presentados en el curso de Bases de Datos I.

Para cada caso de estudio se realiza la identificación de entidades principales del dominio, definición de atributos significativos para cada entidad (simples, compuestos, multivaluados, derivados), establecimiento de atributos identificadores (claves), y diseño de relaciones entre entidades especificando sus cardinalidades (1:1, 1:N, N:M) y participación (total o parcial). El modelo resultante representa la estructura conceptual del sistema antes de su implementación física.

Ejercicio 1: Base de Datos Aerolínea

Enunciado:

Una aerolínea requiere una base de datos relacional para registrar sus operaciones. Las características del negocio son las siguientes:

  • La aerolínea tiene tres recursos principales: Aviones, pilotos y tripulación.
  • Los aviones tienen un código (número de cola), un tipo de avión y una base a donde pertenece y debe regresar cada número de horas.
  • Los pilotos tienen un código, nombre y horas de vuelo.
  • Los miembros de la tripulación tienen un código, nombre y teléfonos.
  • Pilotos y tripulación tienen una base a la que regresan después de cada jornada.
  • Los vuelos van desde un origen a un destino a una hora concreta y tiene un número de vuelo.

Ejercicio 2: Base de Datos Biblioteca

Enunciado:

Crear un diseño E/R que permita gestionar los datos de una biblioteca de modo que:

  • Las personas socias de la biblioteca disponen de un código de socio y además necesitan almacenar su cédula, dirección, teléfono, nombre y apellidos.
  • La biblioteca almacena libros que presta a los socios, de ellos se almacena su título, año en que se escribió el libro, nombre completo del autor (o autores), año en que se editó, en qué editorial fue y el ISBN.
  • Necesitamos poder indicar si un libro en la biblioteca está dañado o no.

Conceptos Aplicados

Entidades: Objetos o conceptos del mundo real
Atributos: Propiedades que describen entidades
Atributos Simples: Valores atómicos indivisibles
Atributos Compuestos: Divisibles en subatributos
Claves Primarias: Identificadores únicos de entidades
Relaciones: Asociaciones entre entidades
Cardinalidad: Número de instancias relacionadas
Participación: Total o parcial en relaciones

Tipos de Cardinalidad Implementadas

1:1 (Uno a Uno)

Una instancia de una entidad se relaciona con exactamente una instancia de otra entidad. Ejemplo: Un piloto tiene una única licencia.

1:N (Uno a Muchos)

Una instancia de una entidad se relaciona con múltiples instancias de otra entidad. Ejemplo: Una base tiene muchos aviones.

N:M (Muchos a Muchos)

Múltiples instancias de ambas entidades pueden relacionarse entre sí. Ejemplo: Pilotos ↔ Vuelos, Libros ↔ Autores.

Aprendizajes Clave

  • • Identificación de entidades a partir de requerimientos del sistema
  • • Definición de atributos relevantes y sus tipos
  • • Establecimiento de relaciones y sus cardinalidades
  • • Distinción entre atributos simples, compuestos y multivaluados
  • • Modelado conceptual independiente de la implementación
  • • Documentación visual mediante diagramas ER
  • • Participación total vs parcial en relaciones
  • • Comprensión del diseño conceptual como base para el modelo lógico
