Library Management System

Sistema de Gestión de Biblioteca en Java

Java Maven MVC DAO Google Drive

Descripción del Proyecto

Sistema completo de gestión de biblioteca desarrollado en Java con arquitectura MVC y persistencia en archivos. Incluye sincronización automática con Google Drive para trabajo colaborativo en equipo, generación profesional de reportes PDF, e implementación de múltiples patrones de diseño.

Características Principales

Gestión completa de usuarios, libros y préstamos (CRUD)
Sistema de búsqueda avanzada con múltiples criterios
Generación de reportes PDF con Apache PDFBox
Sincronización automática con Google Drive
Arquitectura limpia con patrones de diseño
Detección automática de almacenamiento en la nube
Sistema robusto de validación de datos
Interfaz de consola interactiva

Tecnologías Utilizadas

Java 11+ Maven Apache PDFBox 2.0.30 Google Drive API MVC DAO DTO Facade File I/O OOP Advanced

Arquitectura del Sistema

  • View (ViewConsole): Interfaz de usuario por consola
  • Controller (LibraryController): Gestión del flujo de la aplicación
  • Facade (Library): Interfaz simplificada para operaciones complejas
  • DAO (LibraryDAO): Lógica de negocio y acceso a datos
  • Persistence (FileManager): Gestión de archivos y sincronización
  • DTOs: Transferencia de datos entre capas
  • Mappers: Conversión entre diferentes representaciones

Estadísticas del Proyecto

2,500+ Líneas de Código
24 Clases
5 Patrones de Diseño
15+ Archivos

Aprendizajes Clave

  • • Implementación de patrones de diseño en proyectos reales
  • • Persistencia de datos sin frameworks ORM
  • • Manejo de archivos y sincronización en la nube
  • • Generación de PDFs de manera programática
  • • Arquitectura de software escalable y mantenible
  • • Colaboración en equipo con control de versiones
  • • Gestión de dependencias con Maven
  • • Comprensión de limitaciones de archivos vs bases de datos

Desafíos Superados

  • • Implementar sincronización automática con Google Drive sin API oficial
  • • Manejar concurrencia y condiciones de carrera en archivos
  • • Diseñar una arquitectura limpia y desacoplada
  • • Generar PDFs con formato profesional
  • • Coordinar trabajo en equipo sin base de datos centralizada
